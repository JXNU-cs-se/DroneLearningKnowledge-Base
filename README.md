# DroneLearningKnowledge-Base
无人机算法开发与研究学习知识库（持续更新）

团队管理与传承机制建议

为了便于高年级学生更新和低年级学生沿用，建议采用以下机制：

1. **文档即代码**：所有学习笔记、踩坑记录，必须用Markdown编写，提交到团队的GitLab/GitHub **“DroneLearningKnowledge-Base”** 仓库。严禁发Word文档或直接甩链接。
2. **每周“Code Review”与“Paper Sharing”**：低年级展示本周跑的代码结果（哪怕只是画了个框），高年级分享一篇顶会论文（CVPR/ICCV）中与无人机相关的文章。
3. **硬件借阅制度**：将Jetson Nano、Pixhawk飞控等硬件作为“图书馆藏书”出借，必须预约并归还，确保资源不浪费。
4. **版本规划更新机制**：每个学期末，由研二/大四学生牵头，修订本规划文档，将过时的教程剔除（例如某些B站视频已失效），增加当年新出的SOTA模型（如YOLOv11发布，则替换v8作为入门）。
